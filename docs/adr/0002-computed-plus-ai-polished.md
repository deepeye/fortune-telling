# ADR 0002: 八字内容采用"算法计算 + AI 润色"策略

## 状态

已采纳

## 背景

八字命盘有两类内容：确定性计算结果（天干地支、五行生克）和个性化解读文案。纯算法生成文案模板化严重，纯 AI 生成则不可控且可能输出不专业内容。

## 决策

命盘推算用规则引擎严格计算，解读文案先用模板生成框架再用通义千问润色。

## 理由

- **八字计算是确定性数学问题** — 天干地支推算没有歧义，规则引擎保证 100% 准确。
- **纯 AI 生成不可控** — 大模型可能输出荒谬的命理内容，损害"专业感"。
- **纯模板太廉价** — 半信半疑用户需要个性化体验，模板化文案会让用户觉得"每次说的都一样"。
- **折中方案兼顾准确性和个性化** — 计算结果保真，文案有温度有差异。

## 后果

**正向：** 每份报告计算结果一致可信，文案读起来有个性，避免模板廉价感。

**负向：** 需要同时维护规则引擎和 AI 调用链路，系统复杂度增加。每次付费报告生成有 API 费用（通义千问调用）。需要设计 prompt 掤制 AI 输出不偏离命理框架。

**缓解：** 规则引擎输出结构化数据（命盘 JSON），AI prompt 润色时强制引用这些数据作为锚点，避免 AI 自行"发明"命理内容。
