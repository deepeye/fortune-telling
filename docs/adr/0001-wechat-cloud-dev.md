# ADR 0001: 使用微信云开发作为 MVP 技术架构

## 状态

已采纳

## 背景

开发"知命"微信小程序，MVP 阶段需要后端支撑微信登录、云数据库、微信支付和消息推送。可选方案包括微信云开发、自建 Node.js/Python 后端、或 Serverless 方案。

## 决策

使用微信云开发（WeChat Cloud Dev）作为 MVP 技术架构。

## 理由

- **微信登录、支付、订阅消息天然集成** — 这些是产品核心能力（用户绑定、付费解锁、每日运势推送），自建后端需要逐一对接，工作量倍增。
- **零运维、零服务器成本** — MVP 阶段验证产品逻辑，不需要运维负担。
- **审核最顺畅** — 原生微信生态方案，小程序审核无额外障碍。

## 后果

**正向：** MVP 开发速度最快，核心微信能力开箱即用。

**负向：** 绑定微信生态，后续如需发布 H5/App 版本，迁移成本高。云函数冷启动有延迟（约 1-3 秒），AI 文案异步加载策略可缓解此问题。云开发的数据模型和 API 与微信绑定，自建后端迁移需重写数据层。

**缓解：** 产品逻辑层（八字计算引擎、五行评分算法）保持纯 JS 无微信依赖，迁移时只需替换数据层和支付/登录接口。
